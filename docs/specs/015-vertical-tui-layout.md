# SPEC-015: TUI ダッシュボードの縦型レイアウト移行

## 1. 概要
現在の横長 3 ペインレイアウトは、縦長のターミナル環境では視認性が低い。`lazygit` や `lazydocker` のような「サイドバー（リスト） + メイン（詳細）」という縦型レイアウトに刷新し、情報の密度と操作性を向上させる。

## 2. レイアウト設計

### 2.1 画面構成 (2カラム・マルチ行)
画面を左右 2 つの大きなカラムに分け、それぞれをさらに行で分割する。

- **左カラム (Sidebar, 幅 1/3 程度)**:
    - 上段: `Topics` 一覧
    - 下段: `Agents` (Presence) 一覧
- **右カラム (Main Area, 幅 2/3 程度)**:
    - 上段: `Messages` 表示 (メインのスクロール領域)
    - 下段: `Summaries` 表示 (最新の要約)

### 2.2 視覚的演出
- **フォーカス表示**: 現在操作中の領域（ペイン）のボーダー色を鮮やかに変更する（SPEC-008.1 / 014 での合意事項を継承）。
- **タイトル**: 各ペインの左上にタイトルを表示し、境界を明確にする。
- **入力フィールド**: メッセージ投稿時は、画面最下部に全幅の入力ボックスを表示する。

## 3. 実装上の要件
- **パッケージ**: `github.com/charmbracelet/lipgloss` の `JoinHorizontal` および `JoinVertical` を活用。
- **レスポンシブ**: `tea.WindowSizeMsg` を受け取り、ターミナルの高さ・幅に合わせて各ペインのサイズを動的に計算する。
- **スクロール**: メッセージ領域が長くなった場合でも、最新のものが最下部に見えるように配慮する（可能であれば）。

## 4. 実装ステップ
1. `internal/ui/view.go`: レンダリングロジックを横並びから、指定の 2カラム構成に書き換える。
2. `internal/ui/model.go`: 画面サイズ変更のメッセージ処理を確認し、適切なサイズ情報を View に渡す。
3. 動作確認: 縦長のターミナルで各情報が重ならず、見やすく表示されることを確認する。
