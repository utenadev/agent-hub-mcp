# SPEC-010: CIリリースワークフローの修正

## 1. 概要
v0.0.2リリースにおいて、バイナリ（.exe等）がZIP/tar.gzパッケージに含まれない不具合が発生した。原因は、GitHub Actionsのアーティファクト取得時のパス不一致である。

## 2. 原因分析
- `build` ジョブ：`binaries-${os}-${arch}` という名前でプラットフォーム別にアーティファクトをアップロード。
- `release` ジョブ：`merge-multiple: true` を指定して `artifacts/` ディレクトリに一括ダウンロード。
    - これによりディレクトリ構造がフラットになり、全てのバイナリが `artifacts/` 直下に配置される。
- パッケージングスクリプト：`artifacts/binaries-${platform}-${arch}/` というサブディレクトリが存在することを前提に処理を行っていたため、バイナリが見つからず空のパッケージが作成された。

## 3. 修正内容
- `release` ジョブの `download-artifact` ステップにおいて、`merge-multiple: true` を削除（デフォルトの `false` に戻す）。
- これにより、各アーティファクトがそれぞれの名前（`binaries-windows-amd64` 等）のサブディレクトリとして展開され、スクリプトの想定パスと一致するようになる。

## 4. 期待される結果
- 次回のタグプッシュ時、`dist/` ディレクトリに各プラットフォーム（Windows, Linux, macOS）向けのバイナリが含まれたアーカイブが作成され、リリースアセットとして正しくアップロードされる。
