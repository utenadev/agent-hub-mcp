# SPEC-017: TUI ダッシュボードの新しいデザイン

## 1. 概要
`beads_viewer` のレイアウトを参考に、トピック選択を主軸とした洗練されたデザインへ刷新し、操作性（カーソルキー対応）と柔軟な投稿機能を実装する。

## 2. レイアウト設計

### 2.1 トピック選択小窓（Topic Selector）
- **初期表示**: 起動時、画面中央に「トピック選択」の小窓を表示する。
- **機能**: 矢印キーまたは `j/k` でトピックを選択し、`Enter` でメイン画面に遷移。
- **再表示**: メイン画面で `t` キーを押すことで、いつでもこの小窓を呼び出せるようにする。

### 2.2 メイン画面構成
- トピック選択後の内容を表示。
- `beads_viewer` 風の「リスト + 詳細」構成を維持しつつ、視覚的な一貫性（ボーダー、ヘッダー、バッジ等）を向上させる。

## 3. 操作性の向上（キーバインド）
- `h, j, k, l` キー（Vim風）に完全対応。
- **カーソルキー（↑↓←→）** でも同様の操作が可能にする。
- ペイン間の移動（Tab/Shift+Tab）に加え、左右キーでの移動も検討する。

## 4. 投稿機能の強化
- トピックを選択している状態で `p` キーを押すと、投稿フォームを表示する。
- フォーム内で以下のカラムを個別に選択・編集可能にする：
    - **送信者 (Sender)**: デフォルト値を編集可能に。
    - **内容 (Content)**: メッセージ本体。
- 自由に入力・修正ができ、`Enter` で投稿確定、`Esc` でキャンセルの挙動を徹底する。

## 5. 実装タスク（OpenCode向け）
1. `Topic Selector` コンポーネントの実装（初期表示・モーダル表示）。
2. カーソルキー（矢印キー）によるナビゲーションの完全実装。
3. `p` キーによる強化された投稿フォーム（マルチカラム入力）の実装。
4. 全体的なスタイル（`lipgloss`）の `beads_viewer` 風への調整。
